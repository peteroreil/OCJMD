1. Application Overview

The Bodgitt & Scarper Application allows a Customer Service Representative (CSR)
to search and book Subcontractors on behalf of a customer. A CSR interacts with a
client GUI and can search Subcontractors based on the criteria of name and/or location,
retrieve a list of relevant records from a database file, and book the required record.
This application can be run locally or across a network using RMI.

When the user launches the application, regardless of mode, the user will be prompted to
enter configuration details for the first time, e.g. database file, or server port. These
configuration details will be persisted to a properties file (suncertify.properties), where upon
subsequent runs of the application in the same mode, these properties will be used as the default
options for the application.

2. Application Requirements

To run the Bodgitt & Scarper application requires an environment that supports applications
run on Java JRE version 1.7. The Bodgitt & Scarper can run in both standalone or networked
mode. If run in networked mode and in distributed environments, the client and server host systems must be
networked with the specified ports free for use.

The application must be run with the database file db-2x1.db. This file must be available on the
server system if run in networked mode and on the client machine if run in client mode. The application should
only be run with the deliverd database file.

If the application is being run in networked mode, the server side application must be started before the client
application(s) are started.

3. How to Run the Bodgitt & Scarper Application

The Bodgitt & Scarper Application can be started in 3 different modes:

  • Server (Networked)
  • Client (Networked)
  • Client (Standalone)

The jar file runner.jar, can be passed two different flags indicating which of the three modes that the application
will be run in.

"server": indicates that the application will start in Server mode.
"alone": indicates that the application will start in Standalone mode.
If no flag is passed the application will run in the "networked client" mode.

To run the applicaton:
java -jar runner.jar ‹flag›

3.1 Starting the Server in Networked Mode

To allow the application to run in "server" mode, where the application is made available to multiple clients over a
network, the application must be started as follows:

java -jar runner.jar server

Once the server application is started, a configuration dialog will be presented to the user, allowing the user to specify the
location of the database file and to specify the port on which the server will receive remote connections over.
To select the database file, click on the "browse" button and navigate to the db-2x1.db that was supplied with this assignment.
To enter the port number click on the Server Port text field and enter the desired port number. Please enter a port number that is
not currently in use and is avaialble for use. Once all values are entered, press OK to start the server.

Once the server is started a new Window will appear identifying the database file in use and the port that the server componenent is
listening on.

If the user wishes to stop the server, they can close it in one of three ways:

  1. Exit the window
  2. Click Exit
  3. Select File and Exit in the menu bar

3.2 Starting the Client in Networked Mode

To allow the application to run in "networked client" mode, where the application communicates to the server on a specified
host over a designated port, the application must be started as follows:

java -jar runner.jar

After running this command a configuration dialog will be displayed to the user allowing the user to specify the server host ip or
fully qualified domain name of the server host and the port that the server is listening on. Once these have been entered, click OK
and the networked client GUI will be launched.

Note: the server application must be started before the client application.

3.3 Starting the Client in Standalone Mode

To allow the application to run in "standalone" mode, where the application runs in a non-networked mode and connects to a database
file in isolation, the application must be started as follows:

java -jar runner.jar alone

After running this command a configuration dialog will be displayed to the user allowing the user to specify the location of the database file.
To select the database file, click on the "browse" button and navigate to the db-2x1.db that was supplied with this assignment.
Once all values are entered, press OK and the non-networked client GUI will be launched.

4. Using the Client GUI

The client GUI is used the same regardless of whether is is networked or standalone. When the GUI is started, all data is displayed to the user.
The GUI is split into two main sections, the Table and the Interactive (Search & Book Panels).

Searching for a record

To search for all records available, simply hit the "Search" button without providing any name or location search parameters

To search by NAME, enter in a name int the name text arera and press search. (Will return all records that contain that name. See note below).

To search by LOCATION, enter in a location int the location text area and press search. (Will return all records that contain that name.
See note below).

Note:

The Bodgitt & Scarper application implements an OR search condition of either field i.e. it will return all records that
match on Name OR Location.

The application also returns all records that contain a substring of the provided search criteria. i.e. a
recored with name "Bodgitt", will be returned with a name search critera "Bod".

All searches are case sensitive.

Booking a record

To book a record, select the required record in the table so that it becomes highlighted and click the "Book" button. Once selected a dialog will be
presented to the user prompting the user to enter an 8 digit customer ID.
Enter an 8 digit customer ID and click "OK". The Table will then be updated with the provided customer ID value.
